# im-select-ssh.nvim

Automatic IME (Input Method Editor) switching for Neovim over SSH. When you leave insert mode, the current IME is saved and switched to English. When you re-enter insert mode, the previous IME is restored.

Designed for remote Neovim sessions where the local machine runs Windows with a non-English input method (e.g., Chinese, Japanese, Korean).

## How it works

```
Neovim (Lua plugin) --> Rust CLI --> SSH tunnel --> C# client --> im-select.exe
        |                                                             |
   autocommand fires                                        saves/restores IME
   on mode change                                           on local Windows machine
```

Three components cooperate:

1. **Neovim plugin (Lua)** -- hooks `InsertLeave` / `InsertEnter` autocommands, opens a reverse SSH tunnel at startup
2. **Server CLI (Rust)** -- runs on the remote host, sends MsgPack-framed commands through the tunnel
3. **Client (C#)** -- runs on the local Windows machine, listens on the tunnel port and drives [`im-select.exe`](https://github.com/daipeihust/im-select) to save/restore the IME

A randomly generated 6-digit PIN authenticates each session to prevent other processes from issuing IME commands.

## Requirements

| Where | What |
|---|---|
| Remote host | Neovim >= 0.9, Rust toolchain (to build the server CLI) |
| Local machine | Windows, .NET 10 runtime, [`im-select.exe`](https://github.com/daipeihust/im-select) on PATH |
| Both | SSH with key-based auth (the plugin opens a reverse tunnel non-interactively) |

## Installation

### 1. Install the Neovim plugin

Using [lazy.nvim](https://github.com/folke/lazy.nvim):

```lua
{
  "cychen/im-select-ssh.nvim",
  opts = {},
}
```

Or any plugin manager that supports Lua plugins.

### 2. Build the server CLI (on the remote host)

```sh
cd server
cargo build --release
```

Place the resulting binary (`server/target/release/im-select-server`) somewhere on your PATH.

### 3. Build the client (on the local Windows machine)

```sh
cd client
dotnet publish -c Release
```

The published output is in `client/bin/Release/net10.0/publish/`. Run `ImSelectClient` from there.

### Build script (optional)

A convenience script builds all components and collects artifacts into `dist/`:

```sh
python build.py          # release build
python build.py --debug  # debug build
python build.py --clean  # remove dist/ first
```

Use `--skip-server`, `--skip-client`, or `--skip-lua` to build selectively.

## Usage

### Start the client on your local Windows machine

```sh
ImSelectClient --port 9876 --pin <pin>
```

The PIN is generated by the plugin at startup. You can set a fixed PIN in your Neovim config (see below) so the client can be started ahead of time.

### Configure the plugin in Neovim

```lua
require("im-select-ssh").setup({
  -- Path or command name for the Rust CLI (default: "im-select-server")
  server_bin = "im-select-server",

  -- TCP port for the reverse SSH tunnel (default: 9876)
  tunnel_port = 9876,

  -- IME code to switch to on InsertLeave (default: "1033" = en-US)
  default_ime = "1033",

  -- Override the client host IP (default: auto-detected from $SSH_CLIENT)
  client_host = nil,

  -- Fixed PIN for authentication (default: randomly generated each session)
  pin = nil,
})
```

All options are optional. The defaults work for a typical SSH session from a Windows machine.
